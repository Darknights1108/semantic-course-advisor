Project overview

Semantic career search with explainable recommendations

The system models careers, skills, interests, industries, courses, certifications and learning resources as linked entities. Results are produced from graph relationships, not only keyword matching.

Problem

Plain keyword search can retrieve a career title, but it does not explain which skills, learning paths or alternative careers are connected to that result.

Semantic Web Solution

XML source data is converted into RDF triples, queried with SPARQL-style patterns and expanded with OWL-style inference rules.

Demo Evidence

Each tab shows a different part of the knowledge graph: search, recommendation, skill gap planning, visualization, SPARQL and inferred facts.

Semantic Web technologies used

RDF/RDFSCareer data is represented as subject-predicate-object triples and class instances.
SPARQLQuery examples retrieve careers, skills, recommendations and class instances.
OWL/InferenceRules infer alternative-career links, inverse skill facts and prerequisite expansion.
Entity SearchSearch returns career, skill and course entities with graph relationship evidence.
